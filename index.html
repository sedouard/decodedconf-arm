Introducing: Azure Template Language
Parameters 

"parameters": {
  "kafkaNodeIPAddressPrefix": {
    "type": "string",
    "defaultValue": "10.0.0.1",
    "metadata": {
      "description": "The IP address prefix that will be used for constructing a static private IP address for each node in the cluster"
    }
  },
  //...//
  "tshirtSize": {
    "type": "string",
    "defaultValue": "S",
    "allowedValues": [
      "S",
      "M",
      "L"
    ],
    "metadata": {
      "description": "T-shirt size of the Kafka cluster"
    }
  }
}
Define what inputs a template takes. These values can be interpolated later within the template.
Variables 

"variables": {
  "vmStorageAccountContainerName": "vhd",
  "_comment1": "we can reference parameters anywhere!"
  "jumpboxTemplateName": "[parameters('base-name')]",
  "tshirtSizeS": {
    "numberOfZookeeperInstances": 1,
    "numberOfKafkaInstances": 3,
    "vmSize": "Standard_A1"
  },
  "_comment2": "we can set variables, to other variables, too!"
  "numberOfZookeeperInstances": "[variables('tshirtSizeS').numberOfZookeeperInstances]"
}
Like in programming languages, you can define variables for use throughout your template.
Built-in Functions 

"examples": {
  "subscriptionId": "[subscriptionId()]",
  "resourceGroupName": "[resourceGroup().name]",
  "_comment": "twoStrings => 'hellothere'",
  "twoStrings": "[concat('hello', 'there')]",
  "_comment2": "the best way to generate a unique string",
  "uniqueString": "[uniqueString(resourceGroup().name)]"
}

Define the 'stuff' you want to deploy.
Every Resource has These:
An API Version 

"apiVersion": "2015-05-01-preview"
A Resource Type 

"type": "Microsoft.Network/networkInterfaces"
A name label 

"name": "niczoo-1"
The region of the resource 

"location": "westus"
Resource-specific properties 

"properties": {}
But most importantly...
You can Define Dependencies! 

  "dependsOn": [
    "Microsoft.Compute/virtualMachines/anotherVirtualMachine1",
    "Microsoft.Compute/virtualMachines/anotherVirtualMachine2"
  ]
You know how in programming you can do: 

  for (var i in someArray) {
    // do stuff
  }
Or with HTML Templating 

{{#each someArray as element}}
  
    Here is my {{element}}

{{/each}}
We can do the same thing! 

{
    "_comment": "copyIndex() => the current index of iteration"
    "name": "[concat('publicIP', copyindex())]", // eg: publicIP0
    "apiVersion": "[variables('apiVersion')]", 
    "location": "[resourceGroup().location]", 
    "copy": {
        "count": "[variables('numberOfInstances')]", 
        "name": "ipLoop" // we can depend on this loop with this name
    }, 
    "type": "Microsoft.Network/publicIPAddresses", 
    "properties": {
        "publicIPAllocationMethod": "[variables('publicIPAddressType')]"
    }
}
